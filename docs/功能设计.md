# 旅行轨迹记录 APP - 功能设计

> 记录旅行轨迹，生成路线图和行程档案

## 项目定位

一款专注于旅行场景的轨迹记录应用，帮助用户记录旅途足迹，自动串联行程片段，生成可视化路线图和完整的行程档案。

### 目标用户

- 旅行爱好者：记录旅行足迹，生成旅行路线图
- 出差人士：记录出差行程
- 户外探险者：记录徒步、登山轨迹

---

## 核心功能

### 自动打卡

- **触发条件**：在行程日期范围内，每次打开APP自动打卡一次
- **记录内容**：当前位置坐标 + 时间 + 地址
- **标记类型**：自动标记为"自动打卡"，用户可编辑或删除

### 手动打卡

- **触发方式**：用户点击"打卡"按钮
- **记录内容**：位置 + 可选地点名称 + 可选备注
- **适用场景**：到达景点、餐厅、酒店等特定地点时手动记录

### 持续记录

- **触发方式**：用户点击"记录"按钮
- **运动方式**：支持三种 - **自驾**（默认）、**骑行**、**步行**
- **采样频率**：每分钟记录一次定位
- **自动合并**：相同位置自动合并为停留点，记录停留时长
- **数据持久化**：立即写入本地数据库，APP意外退出不丢失
- **适用场景**：
  - 自驾：公路旅行、城市间移动
  - 骑行：骑行游览、环湖骑行
  - 步行：城市漫步、景区游玩

**自动合并逻辑：**
```
每分钟采集位置
      ↓
与上一个点比较距离
      ↓
┌─────┴─────┐
↓           ↓
距离 < 50m   距离 ≥ 50m
↓           ↓
合并到上一个点  创建新点
（更新结束时间）
      ↓
┌─────────────────────────────────────┐
│ 停留点数据结构                       │
│ ├─ 位置：经纬度                      │
│ ├─ 开始时间：10:00                   │
│ ├─ 结束时间：10:35                   │
│ └─ 停留时长：35分钟                  │
└─────────────────────────────────────┘
```

**停留点识别效果：**
| 场景 | 原始数据 | 合并后 |
|------|----------|--------|
| 在景点停留30分钟 | 30个点 | 1个停留点（时长30分钟） |
| 驾车行驶1小时 | 60个点 | 约60个轨迹点（持续移动） |
| 餐厅吃饭1小时 | 60个点 | 1个停留点（时长1小时） |

**运动方式对轨迹补全的影响：**
| 运动方式 | 路线规划 | 采样策略 |
|----------|----------|----------|
| 自驾 | 优先高速公路、国道 | 正常采样 |
| 骑行 | 避开高速，优先骑行道 | 可能增加采样频率 |
| 步行 | 步行路线、小路 | 更密集采样 |

## 使用流程

```
创建行程(日期+目的地)
        ↓
在行程日期内打开APP
        ↓
┌───────┴───────┐
↓               ↓
[自动打卡]     手动操作
每次打开APP    ┌───┴───┐
        ↓      ↓       ↓
    记录位置  [打卡]  [记录]
        ↓     离散点  持续轨迹
        └──────┬──────┘
               ↓
      本地持久化存储
               ↓
      启动时/手动同步到云端
               ↓
      云端处理生成轨迹图
```

**使用场景示例：西藏十日游**

| 日期 | 活动 | 记录方式 |
|------|------|----------|
| 3/15 | 打开APP | 自动打卡：拉萨市区 |
| 3/15 | 逛八廓街 | 持续记录：2小时轨迹 |
| 3/15 | 布达拉宫 | 手动打卡：标记景点 |
| 3/17 | 拉萨→林芝 | 持续记录：沿途轨迹 |
| 3/17 | 米拉山口 | 手动打卡：标记打卡点 |

---

## 核心价值

1. **无感记录** - 打开APP自动打卡，无需手动操作
2. **灵活记录** - 手动打卡 + 持续记录，按需选择
3. **数据安全** - 本地持久化，意外退出不丢数据
4. **智能串联** - 云端自动补全轨迹路线
5. **精美导出** - 生成可分享的轨迹图片

## 数据策略

### 定位采样规则

| 场景 | 触发条件 | 采样频率 | 说明 |
|------|----------|----------|------|
| **自动打卡** | 行程日期内打开APP | 每次启动时1次 | 自动记录当前位置 |
| **手动打卡** | 用户点击打卡 | 触发时1次 | 用户主动操作 |
| **持续记录** | 点击记录按钮 | 每分钟1次 | 持续采集轨迹 |
| 后台运行 | 锁屏/切换APP | 每分钟1次 | 继续采样 |
| APP被杀死 | - | 停止 | 已记录的数据已保存 |

## 异常处理

### GPS定位异常

| 场景 | 处理方式 |
|------|----------|
| GPS信号弱 | 提示"GPS信号弱"，继续记录 |
| GPS不可用 | 提示检查定位权限，暂停记录 |
| 位置跳变 | 过滤异常点（距离突变>500m） |
| 室内无信号 | 提示"当前无法定位"，打卡按钮置灰 |

### APP异常

| 场景 | 处理方式 |
|------|----------|
| 应用被杀死 | 下次打开时提示恢复记录 |
| 手机重启 | 自动检测未完成的记录 |
| 电量不足 | 低电量提示，建议结束记录 |
| 存储空间不足 | 提示清理，禁止开始新记录 |

**恢复记录提示：**
```
┌─────────────────────────────────────┐
```

### 网络异常

| 场景 | 处理方式 |
|------|----------|
| 地图加载失败 | 显示离线占位图，轨迹继续记录 |
| 逆地理编码失败 | 打卡时只显示坐标 |
| 云同步失败 | 仅保存本地，下次有网络时同步 |
